# Chapter Quality Guide — Sarah's Science Curriculum

> **Purpose:** This is the standard every chapter (BFSU + SCI merged) on this site must meet.
> Use it for **new chapter authoring** and as a **retrofit checklist** for existing chapters.
> Aimed at an 8-year-old preparing for **Science Olympiad**-level inquiry.

---

## Part 1 — Two Pedagogical Frameworks

### Framework 1: CEA — Concept → Evidence → Application

Every chapter MUST flow in this order:

1. **CONCEPT (the WHY)** — State the underlying rule or geometry FIRST.
   *Bad:* "Here is a compass."
   *Good:* "Earth is a giant magnet. Every magnet has invisible field lines."

2. **EVIDENCE (the HOW)** — Show how we know the concept is true (multiple sources, not one).
   *Bad:* "The needle points north."
   *Good:* "The needle is a tiny magnet forced to align with Earth's field lines. We can prove it by making our own."

3. **APPLICATION / MYSTERY (the SCIENTIFIC CHALLENGE)** — Use the concept to solve a new puzzle.
   *Bad:* Mystery as a hook with no concept anchor.
   *Good:* "If Earth's magnet is slowly drifting, would your compass still be 100% accurate? How would a scientist check?"

**Rule:** Every mystery must be solvable using a concept explained 30 seconds prior. No riddles without scaffolding.

---

### Framework 2: Three-Tier Mastery (10% / 60% / 30%)

Every quiz and homework set MUST distribute questions roughly in these proportions:

| Tier | % | What it tests | Example |
|------|---|---|---|
| **Tier 1: Vocabulary** | ~10% | Do you know the word? | "What is an axis?" |
| **Tier 2: Application** | ~60% | Can you USE the tool/concept? | "Use a compass to find the chair." |
| **Tier 3: Troubleshooting** | ~30% | What if it breaks? environment changes? | "Why doesn't a compass work at the North Pole?" |

For a 30-question quiz: ~3 vocab / ~18 application / ~9 troubleshooting.
Don't over-index on Tier 1 (memorization) — it's the lowest-intelligence tier.

## Part 2 — The 7 Quality Patterns

Every chapter MUST include all 7:

### 1. "How do we KNOW?" Evidence Slide
- One slide explicitly framed as **evidence**, not facts.
- Show **multiple independent sources** (≥3) that agree.
- Models real scientific verification — multi-source confirmation.

### 2. CER Homework (Claim → Evidence → Reasoning)
- At least **one homework** uses CER structure:
  - **CLAIM:** "I think X is true because..."
  - **EVIDENCE:** "I observed..."
  - **REASONING:** "The scientific rule that explains this is..."

### 3. Spiral Callbacks
- Explicit references to ≥1 **prior chapter**.
- One slide can be a "meta-callback" that ties this chapter to multiple others.
- Format: *"Remember A-5A Magnets? Earth IS a magnet — that's why this works."*

### 4. Graph / Data Interpretation Quiz Question
- At least one quiz Q gives a **data table** or **graph** and asks for prediction or trend.
- Example: shadow length over time → predict noon direction.

### 5. "Explain to a Toddler" Bonus
- One quiz Q (or homework prompt): "Explain X to a 4-year-old in 3 sentences. No big words."
- Tests TRUE understanding vs label memorization.

### 6. "What If?" Experimental Design
- One homework / quiz Q: "What if [variable changes]? What would you predict?"
- Examples: "If Earth spun the opposite way..." / "If we did this in a vacuum..."

### 7. Olympiad Timed Challenge
- One homework with a **time limit** (e.g., "5 minutes to categorize 10 items").
- Builds clutch performance for SciOly station-format events.

## Part 3 — Subject-Specific Olympiad Upgrades

### Phase 1 — Life Science Wrap-Up (Taxonomy, Ecosystems)
- **Dichotomous keys** — student builds their own to identify "5 backyard creatures for a clueless alien."
- **Technical sketching** — circle 3 specific anatomical features that prove classification.
- **Food WEBS** (not chains) with **disturbance quizzes** ("If grass disappears, what happens to the hawk?").

### Phase 2 — Energy (C-1, C-2, C-3, C-4)
- **Energy transformation chains** — Rube Goldberg "Sun → Plant → Human → Kinetic."
- **Variable manipulation** — "If the ruler is shorter, does pitch go up or down?"
- **Household Energy Hunt** — input energy → output energy for 5 household items.

### Phase 3 & 4 — Life Depth + Matter Depth
- **Lab Results questions** — give a data table, ask to predict a 4th measurement.
- **Box & Marble drawings** — draw atoms for solid/liquid/gas.
- **Phase change mysteries** — "Where did the water on the cold glass come from?"
- **Density + concentration** as foundation for chemistry-lab events.

### Phase 5 — Forces (C-5, C-6, C-7)
- **Force-arrow diagrams** — show 3 bridge shapes; draw where forces go.
- **Friction as Energy Thief** — connect back to Phase 2 (kinetic → heat).
- **Mass vs Weight** — calculate planetary weight differences (Mars, Moon, Jupiter).

### Phase 6 — Earth & Space (D-1 through D-8)
- **Spatial reasoning** — model phases of moon with lamp + ball.
- **Model Evaluation** — show a flawed solar system; "act as the judge" and find 3 errors.
- **Map / satellite reading** — identify rivers, deltas, mountain ranges.
- **Rotation vs Revolution** — clearly distinguish.
- **Azimuth degrees** — N=0°, E=90°, S=180°, W=270° + math derivation (360÷4=90°).

## Part 4 — Slide Structure Rules

- **Slide count adapts to depth.** Topic-narrow chapters = 12-15 slides. Topic-rich (multi-curriculum merges) = 18-22 slides. Don't pad; don't truncate.
- **Sessions adapt to depth.** Single-concept chapters = 1 session (~60 min). Two-concept chapters = 2 sessions (e.g. "Why does this happen?" + "How do we measure it?"). Avoid 3 sessions for an 8-year-old unless the topic genuinely demands it.
- **Mysteries (5-6 per chapter)** are interleaved with concept slides, NOT clustered at the end. Each mystery is solvable with a concept already taught.
- **Each chapter must have:**
  - A **conceptual anchor slide** (the WHY before the WHAT).
  - An **evidence slide** (multi-source confirmation).
  - A **spiral callback slide** (1+ link to prior chapter).
  - A **session-end recap** (between Session 1 and Session 2 if multi-session).
  - A **summary slide** (final).

## Part 5 — Homework Standards

Aim for **5-8 homework items** per chapter. Mix of types:

| Type | Purpose |
|---|---|
| **CER Lab** | Claim/Evidence/Reasoning structure |
| **Predict / Observe / Explain** | Multi-trial activity with predictions |
| **Build something** | Hands-on (compass, sundial, balloon rocket) |
| **What if?** | Thought experiment |
| **Tool Selection** | "Which tool fails in [environment]?" |
| **Toddler-Explain Bonus** | Test true mastery |
| **Timed Challenge** | 5-minute Olympiad clutch |
| **Math Integration** | Scale & distance, ratios, division |

## Part 6 — Quiz Standards

**Counts:** 30 questions standard (range 28-35 acceptable).

### Distribution rules:
- ~8 easy (Tier 1 vocabulary) — 25-30%
- ~10 medium (Tier 2 application) — 35%
- ~12 hard (mix of Tier 2 deep and Tier 3 troubleshooting) — 35-40%

### Required question types:
- ≥1 **graph / data table interpretation**
- ≥1 **toddler-explain bonus** (open-ended)
- ≥1 **experimental design** (open-ended or scenario)
- ≥1 **spiral callback** (references prior chapter)
- ≥1 **visual synthesis** (combine 2+ evidence types)
- ≥1 **math-bridge** (forces use of arithmetic to derive answer)
- 5-6 **misconception-flagged** with `misconception: true` and `hints: [...]` arrays
- ≥1 **scenario** (tells a story, asks student to reason)

### Distractor rules:
- **NO "Magic" / "Plants are fake" / "It's random"** distractors.
- **EVERY distractor must be a plausible-but-wrong belief** (a real misconception kids have).
- Correct example: *"Bowling balls are slower because they're bigger"* (wrong but plausible).
- Bad example: *"Bowling balls float because they're full of air"* (obvious nonsense).

### Feedback rules:
- **Every wrong answer's feedback teaches WHY it's wrong.**
- Use **⚠️** marker on wrong-answer feedback.
- Format: `"⚠️ This is a famous misconception (Aristotle thought this 2,300 years ago!). Galileo proved it wrong."`
- Don't just say "No" — explain.

### Hints (for misconception questions):
- 2 progressive hints per misconception question.
- Hint 1: gentle nudge toward thinking the right way.
- Hint 2: more specific, almost gives the answer.
- Format: `hints: [hint1, hint2]` array on the question object.

## Part 7 — Quality Verification Checklist

Before pushing, every chapter MUST pass:

### Content checks
- [ ] All 7 quality patterns present (Evidence slide, CER, Spiral, Graph, Toddler, What-if, Timed)
- [ ] CEA flow (Concept → Evidence → Application) — no riddle-first jumps
- [ ] Three-Tier Mastery distribution (10/60/30)
- [ ] 5-6 mysteries interleaved (not clustered)
- [ ] ≥1 spiral callback to a prior chapter
- [ ] All distractors are plausible misconceptions

### Technical checks
- [ ] **JS validates** (`new Function(scriptContent)` doesn't throw)
- [ ] **Slide count matches `TOTAL_SLIDES`** (and "1 / N" counter)
- [ ] **Video oEmbed AND embed-allowed** (both `oembed=200` AND no `PLAYABILITY_ERROR_CODE_EMBEDDER` in `/embed/<id>`)
- [ ] **Video has fallback link** to YouTube even if embed works
- [ ] All `data-slide` numbers are unique and sequential

### Template-leak checks (5 things to verify per chapter)
- [ ] **`<title>` tag** matches chapter title (not "Energy to Run the Body")
- [ ] **Chapter-header `<h1>`** is chapter-specific
- [ ] **Chapter-header `<p class="lead">`** is chapter-specific intro
- [ ] **Parts-preview cards** describe THIS chapter (not B-9 sandwich)
- [ ] **Quiz intro** says actual count + chapter theme (not "30 questions on body energy")
- [ ] **Homework download filename** uses `${PFX}` not `B9_`

## Part 8 — Process for New Chapters

1. **Plan first** (don't dive in):
   - What is the BIG question?
   - What is the CORE concept (geometric, physical, biological)?
   - What 5-6 mysteries does that concept naturally generate?
   - What spiral callbacks make sense?
   - What's the right slide count + session count for this depth?

2. **Find a verified video**: search → check `oembed=200` AND `/embed/<id>` does NOT contain `PLAYABILITY_ERROR_CODE_EMBEDDER`. If embed-blocked, still use it but include a clickable YouTube link as fallback.

3. **Write the spec** in `outputs/gen/<chapter>.py` — slide content, homework, quiz.

4. **Build and finalize**:
   ```python
   write_chapter(SPEC)
   finalize_chapter(out_path)  # auto-applies hints, quiz UX, fix_quiz_intro, fix_chapter_header
   ```

5. **Run verification** against the checklist above.

6. **Push to GitHub** with a clear commit message documenting which patterns were used.

7. **User reviews. Iterate.**

## Part 9 — Process for Retrofitting Existing Chapters

For each existing chapter, ask:

1. **Is there a conceptual anchor?** (If not, add one as slide 3)
2. **Are mysteries solvable with concepts already taught?** (If not, reorder or add concept)
3. **Is there a graph/data Q?** (If not, add)
4. **Is there a toddler-explain Q?** (If not, add)
5. **Is there a "What if?" prompt?** (If not, add)
6. **Are distractors plausible?** (If "Magic" appears, rewrite)
7. **Are misconception Qs flagged with hints?** (If not, add)
8. **Spiral callbacks present?** (If not, add)
9. **All template leaks fixed?** (Check the 5 leak points)

See `RETROFIT_PLAN.md` for the prioritized chapter list.

## Appendix — Tooling

These post-processors live in `outputs/gen/` and are called by `finalize_chapter()`:

- `add_hints.py` — installs hint feature CSS + JS into a chapter
- `upgrade_quiz_ux.py` — adds quizSkipped/quizFinished states + selected/correct CSS
- `fix_quiz_intro.py` — replaces "30 questions on body energy." with chapter-appropriate text + fixes hw download filename
- `fix_chapter_header.py` — replaces leaked B-9 chapter-header with content derived from `<title>` + slide #2

Build a chapter:
```python
from build_chapter import write_chapter, finalize_chapter
write_chapter(SPEC)  # generates from b9 template + spec
finalize_chapter(out_path)  # auto-applies all 4 post-processors
```
